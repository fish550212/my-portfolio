Lockstep
Key Lockstep technology
  • Multiple-instruction
    • Detects hard-to-detect sequential control path bugs
    • Self-consistency technology
  • Single-instruction
    • Detects datapath bugs
    • Exhaustive set of single-instruction checkers
Functional and formal coverage confirmed using 3rd party tool
Implementation
    • The Lockstep technology is implemented in an SV module used for verification only
      Single- and multiple-instruction checkers
    • Used in conjunction with any commercial Formal Bounded Model Checking (BMC) tool
    • Implementation independent, does not require microarchitecture or design knowledge
Multiple-instruction verification
Detects complex, sequential control path bugs
  • Multiple transformations ensure automated detection of wide range of control path bugs
  • Error detection using duplicated instructions for validation Data forwarding error
    Control flow checking using software signatures for validation Branch error
    Unexpected kill/flush event Illegal instruction
    Control flow tracking using software signatures for validation System hang
    Proactive load and check Load/store error
  • Accurate:Detects actual bugs
  • Ready-made constraints: Avoids masked bugs
  • Implementation independent
  • Short debug trace
Multiple-instruction verification using self-consistency
    • Formal tool (BMC) creates exhaustive set of base instructions
    • Depth of instruction sets customizable by user
    • Lockstep module duplicates each set of base instructions
    • Base instructions use one half of memory/register file
    • Duplicate set the other half
    • Result pairs compared at short intervals
    • Expect same results → self-consistency
    • Very short error detection latency

Coverage analysis
  • Functional coverage
    • Exhaustive multiple-instruction coverage
      • Across sequence combinations
      • E.g.: 65,536 / 65,536 possible sequential conditions covered by subset of 4 instructions and depth of 8
    • Exhaustive single-instruction coverage
      • Across instruction types, source operands, destination operands
      • E.g.: 4,544/4,544 conditions on MUL instruction
  • Formal coverage
    • Code overage of single- and multiple- instruction sequences on target RISC-V implementation
    • E.g. ZERO-RISCY
      • Stimuli coverage: 90.32%
      • COI coverage: 98.05%
      • Proof core coverage: 96.71%
